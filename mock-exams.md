# კომპიუტერის არქიტექტურა — ფინალის სავარჯიშო ვარიანტები

**ფორმატი:** 5 ღია (წერითი) კითხვა · სულ **30 ქულა** · თითო კითხვა **6 ქულა**
**ლოგიკა:** ვარიანტები აგებულია ლექტორის ნიმუშზე — შედარებები, დასახელებული ტექნოლოგიები, **ორნაწილიანი (combo) კითხვები** და თითო **გამოთვლა** ვარიანტში.
**წყარო:** allweeks.pdf (თავები 8–17). სრული პასუხები იხ. `exam-prediction.md`.

> **ნიშნული:** 🔗 = combo (ორი კითხვა ერთში, ქულა იყოფა 3+3) · 🧮 = გამოთვლა
> ჯერ სცადე დახურული წიგნით, შემდეგ შეამოწმე ბოლოს მოცემული პასუხების გასაღებით.

1. 🔗 **(3+3 ქ.)** აღწერეთ მონაცემების **ერთგამტარიანი** გადაცემის პრინციპი **და** **დიფერენციალური** გადაცემის პრინციპი. *(თ.10)*
2. **(6 ქ.)** შეადარეთ ერთმანეთს **USB 2.0 და USB 3.0** სტანდარტები გადაცემის სიჩქარისა და სიმძლავრის კუთხით. *(თ.12)*
3. 🔗 **(3+3 ქ.)** აღწერეთ ფერების რეჟიმები **Hi-Color და Real Color** **და** რეჟიმი **True Color**. *(თ.17)*
4. **(6 ქ.)** ჩამოაყალიბეთ DDR4 სტანდარტით გათვალისწინებული **შეცდომების აღმოჩენისა და კორექციის** მეთოდები. *(თ.9)*
5. 🧮 **(6 ქ.)** პროცესორის სიმძლავრეა **120 ვატი**. გამოთვალეთ ძაბვის რეგულატორზე მისაწოდებელი დენი 12 ვ ძაბვისა და 80% მ.ქ.კ.-ის შემთხვევაში. *(თ.13)*

# ✅ პასუხების გასაღები (მოკლე)

## ვარიანტი 1
1. **ერთგამტარიანი:** ყოველი სიგნალი ორი გამტარით — ერთით რეალური ძაბვა, მეორე დამიწებული; იაფი, მაგრამ დაბალი შეფერხებამედეგობა და მოკლე სიგრძე. **დიფერენციალური:** ორი გამტარი — პირდაპირი + ინვერსიული სიგნალი; ხმაური ორივეზე ერთნაირად მოქმედებს, მიმღები სიგნალს სხვაობით აღადგენს → მეტი მედეგობა, მეტი სიგრძე/სიხშირე.
2. USB 2.0 → max 480 მბიტ/წმ, max დენი 500 მა; USB 3.0 → max 4.8 გბიტ/წმ, max დენი 900 მა.
3. **Hi-Color:** 2¹⁵ = 32768 ფერი, 15 ბიტი/პიქსელი. **Real Color:** 2¹⁶ = 65536 ფერი, 16 ბიტი/პიქსელი (მწვანეს დამატებითი ბიტი თვალის მგრძნობელობის გამო).
4. მისამართები/ბრძანებები → ლუწობის კონტროლი; ჩასაწერი მონაცემები → CRC კოდები; ასევე ECC-მხარდამჭერი DDR4 მოდულები.
5. 🧮 I = 120 / (12 × 0.8) = **12.5 ა**.

## ვარიანტი 2
1. SATA მიმდევრობითია → ნაკლები გამტარი ვიდრე პარალელურ ATA-ში; მარტივი/იაფი კაბელი, უკეთესი ვენტილაცია, ფაზური ძვრის თავიდან აცილება.
2. **FAT32:** 32-თანრიგა დამისამართება, ტომი max 2 ტბ. **exFAT:** Microsoft, 2006, ფლეშისთვის, 4 გბ ფაილის ლიმიტი არ აქვს. **NTFS:** Microsoft-ის სტანდარტი — წვდომის კონტროლი, კვოტები, დაშიფვრა, კომპრესია, მეტამონაცემები, საიმედოობა.
3. PCI — პარალელური, ერთმიმართულებიანი, ბევრი გამტარი; PCI Express — მიმდევრობითი, დუპლექსური, ტრასებით, ბევრად მაღალი გამტარუნარიანობა.
4. **Power Good:** ადასტურებს, რომ ძაბვები ნორმაშია; დარღვევისას ითიშება და პროცესორი გადაიტვირთება. **Power_On:** პლატიდან კვების ბლოკზე — პროგრამული გამორთვა/კლავიატურით ჩართვა. **+5V Standby:** მცირე სიმძლავრის კვება გამორთულ მდგომარეობაშიც.
5. 🧮 2500 × 8 × 0.8 = 16000 მბიტ/წმ = 2000 მბაიტ/წმ = 2 გბ/წმ ერთი მიმართულებით → დუპლექსი **4 გბ/წმ**.

## ვარიანტი 3
1. 2 ტბ-ზე დიდი დამგროვებლები, 128-მდე ლოგიკური განყოფილება, Secure Boot, სენსორული კონტროლი, გრაფიკული ინტერფეისი + მაუსი, რამდენიმე ენა, UEFI დრაივერები, OS-ის სწრაფი ჩატვირთვა.
2. **Fast Memory Access:** მეხსიერებასთან მუშაობის ოპტიმიზაცია — ბრძანებების რიგითობას აანალიზებს, შეთანხმებადებს თანმიმდევრულად ასრულებს, წაკითხვის შეზღუდვას ამცირებს. **Matrix Storage:** RAID მასივებთან მუშაობის ტექნოლოგია.
3. DDR3 Multi-Drop — ერთ არხს რამდენიმე მოდული (ზღუდავს წარმადობას); DDR4 Point-to-Point — ერთი მოდული ერთ არხზე, პარალელური მიმართვა, მოდულების დამატებისას სწრაფქმედება იზრდება.
4. **CD-R:** ერთჯერადი ჩაწერა — ლაზერი ორგანულ საღებავს 250–300°C-მდე წვავს, მონაკვეთი გაუმჭვირვალდება. **CD-RW:** მრავალჯერადი — პოლიკრისტალური ფენა ოპტიკურ თვისებებს მრავალჯერ იცვლის (წაშლა/ჩაწერა).
5. **SMART** (Self-Monitoring, Analysis & Reporting) — HDD-ის მდგომარეობის შეფასება/შეცდომების წინასწარმეტყველება; დაზიანების ალბათობისას BIOS-ს/სისტემას ატყობინებს. **აანალიზებს:** თავაკის დაშორებას, HDD↔Cache სიჩქარეს, bad სექტორებს, ბრუნვებს, თავაკის გადაადგილებებს, დატრიალების დროს, მოძებნის შეფერხებებს, დაკალიბრებებს.

## ვარიანტი 4
1. DRAM — იაფი, მაღალი ინტეგრაცია, ძირითადი RAM, მაგრამ ნელი + რეგენერაცია; SRAM — სწრაფი, რეგენერაცია არ სჭირდება, მაგრამ ძვირი, ნაკლები ტევადობა → Cache.
2. **ტექსტური:** ეკრანი სტრიქონებად/სვეტებად, თითო უჯრედში ერთი სტანდარტული სიმბოლო (მაგ. ბრძანებითი სტრიქონი). **გრაფიკული:** თითო პიქსელი ცალკე იმართება → ნებისმიერი ფორმა/ფერი, მაგრამ მეტი რესურსი.
3. DDR — მონაცემები სინქროსიგნალის ორივე ფრონტზე (წინა+უკანა) → ერთ ციკლში ორი მონაცემი → ორმაგი სიჩქარე.
4. **Core i:** მეხსიერების კონტროლერი ჩაშენებულია პროცესორში (2+ არხი), ხშირად ჩაშენებული GPU → North Bridge/MCH ზედმეტი ხდება. **I/O Controller Hub:** South Bridge-ის ახალი სახელი — USB/ATA/I/O ინტერფეისები, LPC-ით Firmware Hub/BIOS და Super I/O.
5. 🧮 I = 300 / 12 = **25 ა**. 10 ა ტერმინალებით: 2 ხაზი → 20 ა (ვერ კმარა), 3 ხაზი → 30 ა → საჭიროა მინ. **6-კონტაქტიანი PCI Express** გასართი სამი +12ვ კონტაქტით.

## ვარიანტი 5
1. **დანიშნულება:** პროგრამების/მონაცემების შენახვა; მუდმივი მეხსიერება კვებისგან დამოუკიდებელია — გამორთვის შემდეგაც ინახავს ინფორმაციას. **ჩიპების ტიპები:** ROM (დამზადებისას, არ იცვლება) → PROM (ერთჯერადი) → EPROM (მრავალჯერ, იშლება UV/რენტგენით) → EEPROM/Flash (პროგრამული წაშლა/განახლება — თანამედროვე სტანდარტი).
2. მიმდევრობითი უნივერსალური ინტ.; დუპლექსი, ტრასები/ზოლები ×1/×2/×4/×8/×16; PCI-თან შეთავსება, ჩაშენებული სინქრონიზაცია, მცირე მოლოდინის დრო, ცხელი კომუტაცია, კვების მართვა.
