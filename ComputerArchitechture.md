# 1. BIOS, UEFI, CMOS, POST და ROM

## 1.1 BIOS

### განმარტება

**BIOS** ნიშნავს **Basic Input-Output System**-ს, ანუ საბაზისო შეყვანა-გამოყვანის სისტემას.

BIOS არის firmware, რომელიც ინახება დედაპლატაზე არსებულ არამდგრად მეხსიერებაში და მუშაობას იწყებს კომპიუტერის ჩართვისთანავე.

### BIOS-ის მთავარი ფუნქციები

| ფუნქცია | ახსნა |
| --- | --- |
| POST | კომპიუტერის ძირითადი მოწყობილობების შემოწმება |
| BIOS Setup | სისტემური პარამეტრების კონფიგურაცია |
| Boot | ოპერაციული სისტემის ჩატვირთვის დაწყება |
| I/O მომსახურება | დაბალი დონის შეყვანა-გამოყვანის ოპერაციები |

### BIOS-ის ცნობილი მწარმოებლები

- AMI
- Award Software
- Phoenix Technologies
- Microid Research

### გამოცდისთვის მნიშვნელოვანი

BIOS მუშაობს **ოპერაციული სისტემის ჩატვირთვამდე**. ამიტომ თუ ოპერაციული სისტემა ჯერ არ არის ჩატვირთული, მოწყობილობების საწყის ინიციალიზაციას BIOS ასრულებს.

---

## 1.2 BIOS-ის კომპონენტები

BIOS-ის ძირითადი კომპონენტებია:

| კომპონენტი | დანიშნულება |
| --- | --- |
| POST | აპარატურის შემოწმება |
| BIOS Setup | პარამეტრების დაყენება |
| Bootstrap Loader | ჩასატვირთი მოწყობილობის მოძებნა და OS-ის ჩატვირთვის დაწყება |
| BIOS Drivers / Routines | საბაზისო მოწყობილობებთან ურთიერთობა |

---

## 1.3 CMOS Setup

### განმარტება

**CMOS Setup** არის პარამეტრების კონფიგურაციის გარემო, სადაც ინახება BIOS-ის მიერ გამოყენებული სისტემური პარამეტრები.

### CMOS Setup-ში ინახება

- თარიღი და დრო
- Boot order
- დისკების კონფიგურაცია
- მოწყობილობების ჩართვა/გამორთვა
- პროცესორისა და მეხსიერების ზოგიერთი პარამეტრი
- დედაპლატის სხვა სისტემური პარამეტრები

### CMOS vs BIOS

| ცნება | მნიშვნელობა |
| --- | --- |
| BIOS | პროგრამა / firmware, რომელიც იწყებს სისტემას |
| CMOS Setup | პარამეტრების კონფიგურაციის ადგილი |
| CMOS Memory | მეხსიერება, სადაც ტრადიციულად პარამეტრები ინახებოდა |
| CMOS Battery | კვება, რომელიც პარამეტრებს ინარჩუნებს გამორთულ მდგომარეობაში |

---

## 1.4 POST

### განმარტება

**POST** ნიშნავს **Power-On Self-Test**-ს.

ეს არის თვითტესტირების პროცესი, რომელსაც BIOS ან UEFI ასრულებს კომპიუტერის ჩართვისას.

### POST ამოწმებს

- პროცესორს
- RAM-ს
- ვიდეოსისტემას
- კლავიატურას
- დისკურ მოწყობილობებს
- დედაპლატის ელემენტებს
- გაფართოების ბარათებს

### POST-ის შეცდომების შეტყობინება

| მეთოდი | გამოყენება |
| --- | --- |
| ეკრანზე შეტყობინება | თუ ვიდეო სისტემა მუშაობს |
| ხმოვანი სიგნალი / Beep Code | თუ ეკრანზე გამოტანა შეუძლებელია |
| დიაგნოსტიკური კოდი | ზოგიერთ დედაპლატაზე |

### დასამახსოვრებელი

**POST = Power On → Self Test**

თუ ეკრანი არ მუშაობს, შეცდომა ხშირად ხმოვანი სიგნალებით გადმოიცემა.

## 1.5 BIOS-ის ჩატვირთვის პროცესი

კომპიუტერის ჩართვისას პროცესი ასეთია:

1. კვება მიეწოდება სისტემას.
2. BIOS / UEFI იწყებს მუშაობას.
3. POST ამოწმებს აპარატურას.
4. BIOS კითხულობს CMOS პარამეტრებს.
5. ირჩევა ჩასატვირთი მოწყობილობა.
6. იწყება bootloader-ის ან OS loader-ის გაშვება.
7. იტვირთება ოპერაციული სისტემა.

## 1.6 BIOS vs UEFI

### UEFI-ის განმარტება

**UEFI** ნიშნავს **Unified Extensible Firmware Interface**-ს.

ეს არის BIOS-ის თანამედროვე შემცვლელი firmware გარემო.

### BIOS და UEFI შედარება

| მახასიათებელი | BIOS | UEFI |
| --- | --- | --- |
| ასაკი | ძველი სტანდარტი | თანამედროვე სტანდარტი |
| ინტერფეისი | ძირითადად ტექსტური | შეიძლება იყოს გრაფიკული |
| დისკის მხარდაჭერა | შეზღუდულია დიდი დისკებისთვის | უკეთ უჭერს მხარს დიდ დისკებს |
| Partition სტილი | MBR | GPT |
| უსაფრთხოება | შეზღუდული | Secure Boot მხარდაჭერა |
| ჩატვირთვა | Legacy boot | თანამედროვე boot manager |
| გაფართოებადობა | ნაკლები | მეტი |

### UEFI-ის უპირატესობები

- უფრო დიდი დისკების მხარდაჭერა
- GPT partition table
- Secure Boot
- უფრო მოქნილი ჩატვირთვის პროცესი
- უფრო თანამედროვე ინტერფეისი
- უფრო სწრაფი და გაფართოებადი firmware გარემო

---

## 1.7 ROM მეხსიერების ტიპები

BIOS / UEFI ინახება არამდგრად მეხსიერებაში.

### ROM ტიპების შედარება

| ტიპი | სრული სახელი | გადაპროგრამება | წაშლის / ჩაწერის მეთოდი |
| --- | --- | --- | --- |
| ROM | Read-Only Memory | არა | ქარხნულად იწერება |
| PROM | Programmable ROM | ერთხელ | მომხმარებელი წერს ერთხელ |
| EPROM | Erasable PROM | დიახ | ულტრაიისფერი სხივით იშლება |
| EEPROM | Electrically Erasable PROM | დიახ | ელექტრულად იშლება |
| Flash ROM | Flash Read-Only Memory | დიახ | ბლოკებად ელექტრულად იშლება და იწერება |

### მეხსიერების ხრიკი

- **PROM** — Programmable once
- **EPROM** — Erased by light
- **EEPROM** — Electrically erased
- **Flash ROM** — თანამედროვე BIOS/UEFI firmware-ის ხშირი ფორმა

### გამოცდის კითხვა

## 2.3 SRAM

### განმარტება

**SRAM** ნიშნავს **Static Random Access Memory**-ს.

SRAM მონაცემს ინახავს ტრანზისტორული trigger / flip-flop სტრუქტურით.

### მახასიათებლები

| მახასიათებელი | SRAM |
| --- | --- |
| Refresh სჭირდება? | არა |
| სიჩქარე | ძალიან სწრაფი |
| ფასი | ძვირი |
| სიმჭიდროვე | დაბალი |
| გამოყენება | Cache მეხსიერება |

---

## 2.4 DRAM

### განმარტება

**DRAM** ნიშნავს **Dynamic Random Access Memory**-ს.

DRAM მონაცემს ინახავს კონდენსატორში მუხტის სახით.

### მახასიათებლები

| მახასიათებელი | DRAM |
| --- | --- |
| Refresh სჭირდება? | დიახ |
| სიჩქარე | SRAM-ზე ნელი |
| ფასი | იაფი |
| სიმჭიდროვე | მაღალი |
| გამოყენება | ძირითადი RAM |

### რატომ სჭირდება DRAM-ს Refresh?

კონდენსატორში მუხტი დროთა განმავლობაში იკარგება, ამიტომ მონაცემის შესანარჩუნებლად აუცილებელია პერიოდული განახლება.

---

## 2.5 SRAM vs DRAM

| მახასიათებელი | SRAM | DRAM |
| --- | --- | --- |
| სრული სახელი | Static RAM | Dynamic RAM |
| მონაცემის შენახვა | Flip-flop / ტრანზისტორები | კონდენსატორი + ტრანზისტორი |
| Refresh | არ სჭირდება | სჭირდება |
| სიჩქარე | სწრაფი | შედარებით ნელი |
| ფასი | ძვირი | იაფი |
| სიმჭიდროვე | დაბალი | მაღალი |
| გამოყენება | Cache | ძირითადი ოპერატიული მეხსიერება |

### დამახსოვრება

- **S = Stable** — SRAM-ს refresh არ სჭირდება.
- **D = Drains** — DRAM-ში მუხტი იკარგება.

---

## 2.6 Cache მეხსიერება

### განმარტება

**Cache** არის მცირე მოცულობის, სწრაფი მეხსიერება, რომელიც ინახავს ხშირად გამოყენებულ მონაცემებსა და ინსტრუქციებს.

### მთავარი იდეა

CPU-ს RAM-ზე ლოდინი რომ არ მოუწიოს, ხშირად გამოყენებული მონაცემები ინახება უფრო სწრაფ Cache-ში.

### Cache-ის მახასიათებლები

- ჩვეულებრივ აგებულია SRAM-ზე
- RAM-ზე სწრაფია
- ამცირებს CPU-ის ლოდინის დროს
- ინახავს ხშირად ან ახლახან გამოყენებულ მონაცემებს

---

## 2.7 DIP Switch

### განმარტება

**DIP switch** არის პატარა ფიზიკური გადამრთველების ჯგუფი, რომელიც ძველ სისტემებში გამოიყენებოდა აპარატურული პარამეტრების ხელით დასაყენებლად.

### გამოყენება

- რეჟიმის არჩევა
- მისამართის კონფიგურაცია
- მოწყობილობის პარამეტრების დაყენება
- დედაპლატის ან გაფართოების ბარათის კონფიგურაცია

---

# 3. DDR მეხსიერებები: DDR, DDR2, DDR3, DDR4, DDR5

## 3.1 DDR-ის განმარტება

**DDR** ნიშნავს **Double Data Rate**-ს.

DDR მეხსიერება მონაცემებს გადასცემს clock სიგნალის ორივე ფრონტზე:

- rising edge
- falling edge

ამიტომ ერთ clock ციკლში მონაცემის გადაცემა ორჯერ ხდება.

---

- გამტარუნარიანობა
- ენერგოეფექტურობა
- სამუშაო ძაბვა
- prefetch ზომა
- მოდულის მაქსიმალური მოცულობა
- სიგნალიზაცია
- სალტის არქიტექტურა

# 5. PCI და PCI Express

## 5.1 PCI

### განმარტება

**PCI** არის ძველი პარალელური გაფართოების სალტე.

### მახასიათებლები

- პარალელური გადაცემა
- რამდენიმე მოწყობილობა ერთ სალტეზე
- PCI Express-მდე ფართოდ გამოიყენებოდა
- შედარებით დაბალი გამტარუნარიანობა

---

## 5.2 PCI Express

### განმარტება

**PCI Express / PCIe** არის თანამედროვე, მაღალსიჩქარიანი, სერიული, point-to-point გაფართოების ინტერფეისი.

### მახასიათებლები

- სერიული გადაცემა
- Point-to-point კავშირი
- Lane-ების გამოყენება
- შესაძლებელია x1, x2, x4, x8, x16 კონფიგურაციები
- მაღალი გამტარუნარიანობა
- გამოიყენება ვიდეო ბარათებისთვის, SSD-ებისთვის, ქსელური ბარათებისთვის და სხვ.

---

## 5.3 PCI vs PCI Express

| მახასიათებელი | PCI | PCI Express |
| --- | --- | --- |
| გადაცემა | პარალელური | სერიული |
| კავშირი | საერთო სალტე | Point-to-point |
| სიჩქარე | დაბალი | მაღალი |
| გაფართოება | შეზღუდული | x1, x4, x8, x16 |
| თანამედროვე გამოყენება | იშვიათი | ფართოდ გამოიყენება |

- ერთი მისაღებად

## 8.13 კვების გამოთვლის ფორმულა

დენის გამოსათვლელად:

```
I = P / (V × η)
```

სადაც:

- I = დენი ამპერებში
- P = სიმძლავრე ვატებში
- V = ძაბვა
- η = ეფექტურობა

### მაგალითი 1

მოცემულია:

- სიმძლავრე: 120 W
- ძაბვა: 12 V
- ეფექტურობა: 80% = 0.8

```
I = 120 / (12 × 0.8) = 12.5 A
```

### მაგალითი 2

```
I = 75 / (12 × 0.8) = 7.8 A
```

### მაგალითი 3

```
I = 300 / 12 = 25 A
```

### მაგალითი 4

```
I = 250 / (12 × 0.8) ≈ 20.83 A
```

იდეა იყო ხშირად გამოყენებული მონაცემების Flash მეხსიერებაში შენახვა.

## 10.2 Zone Bit Recording

### განმარტება

**Zone Bit Recording** არის ტექნოლოგია, რომლის დროსაც გარე ზონებში მეტი სექტორი თავსდება, ვიდრე შიდა ზონებში.

### რატომ?

გარე ბილიკი უფრო გრძელია, ამიტომ მასზე მეტი მონაცემის მოთავსება შეიძლება.

---

## 10.3 Average Seek Time

### განმარტება

**Average Seek Time** არის საშუალო დრო, რომელიც სჭირდება წაკითხვის / ჩაწერის თავს საჭირო track-მდე მისასვლელად.

---

## 10.4 Access Time

### განმარტება

**Access Time** არის დრო, რომელიც საჭიროა მონაცემზე წვდომისთვის.

ჩვეულებრივ მოიცავს:

- seek time
- rotational latency
- controller overhead

---

## 10.5 მონაცემთა გადაცემის სიჩქარე

ერთ-ერთი ფორმულა:

```
MDTR = SPT × 4096 × RPM / 60
```

სადაც:

- MDTR = მაქსიმალური მონაცემთა გადაცემის სიჩქარე
- SPT = sectors per track
- 4096 = სექტორის ზომა ბაიტებში
- RPM = ბრუნი წუთში

---

## 10.6 HDD Cache

HDD Cache არის შიდა ბუფერული მეხსიერება, რომელიც დროებით ინახავს მონაცემებს.

### დანიშნულება

- წაკითხვის დაჩქარება
- ჩაწერის ოპტიმიზაცია
- დისკის მექანიკური დაყოვნებების შემცირება

## 10.7 SMART

### განმარტება

**SMART** ნიშნავს **Self-Monitoring, Analysis and Reporting Technology**-ს.

ეს არის ტექნოლოგია, რომელიც აკვირდება დისკის მდგომარეობას.

### SMART აკვირდება

- შეცდომების რაოდენობას
- bad sector-ებს
- ტემპერატურას
- spin-up დროს
- read/write შეცდომებს
- დისკის ზოგად ჯანმრთელობას

### გამოცდისთვის

SMART გამოიყენება დისკის შესაძლო დაზიანების წინასწარ გამოსავლენად.

# ერთგვერდიანი საბოლოო გამეორება

## Firmware

- **BIOS** — ძველი firmware, იწყებს კომპიუტერს.
- **UEFI** — BIOS-ის თანამედროვე შემცვლელი; GPT, Secure Boot, დიდი დისკები.
- **POST** — აპარატურის შემოწმება ჩართვისას.
- **CMOS Setup** — BIOS-ის პარამეტრების დაყენება.
- **Flash ROM** — თანამედროვე BIOS/UEFI-ის ჩიპი.

| --- | --- |
| PROM | ერთხელ იწერება |
| EPROM | UV სინათლით იშლება |
| EEPROM | ელექტრულად იშლება |
| Flash | ელექტრულად, ბლოკებად |

## RAM

- **SRAM** — სწრაფი, ძვირი, refresh არა, Cache.
- **DRAM** — იაფი, refresh სჭირდება, ძირითადი RAM.
- **Register** — CPU-ის შიგნით ყველაზე სწრაფი მეხსიერება.
- **Cache** — SRAM-ზე აგებული სწრაფი შუალედური მეხსიერება.
